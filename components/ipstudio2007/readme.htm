InfoPower Studio 2007 –5/15/07 Version 5.0.3.4

 

System Requirements

Delphi 2007, Builder 2007, Delphi 2006, Builder 2006, Delphi 7

 

Compatibility issues of InfoPower Studio/Win32 with InfoPower 4000:

InfoPower Studio 2007 for Win32 is designed to be 100% backwards compatible with InfoPower 4000 and InfoPower Studio 2006.  No changes are required in your project.

 

For those who are using TwwDBRichEditMSWord, make sure you add the \CodeGear\Rad Studio\5.0\imports directory to your library search path so that the wordxp.dcu or word2000.dcu references are resolved.

 

New Features in 2007:

 

InfoPower Studio 2007 adds the following new functionality.

 

Vista compatibilty:

·         InfoPower dialogs and controls enhanced to support the vista look and feel.

 

Data Validation enhancements:

·         Regular expression support for data entry validation. Regular expressions can be used with the TwwDBEdit, TwwDBGrid, and TwwDataInspector components. InfoPower 2007 also includes a library of ready-to-go common regular expressions for data validation. See the TwwDBEdit.RegexMask property and its related documentation.

·         New centralized visual manager for data validation: Developer can globally assign validation background and foreground colors for invalid data entered by the user. Invalid data is determine by either regular expressions or picture mask validation. See the TwwIntl.VisualFeedback property.

 

RichEdit enhancements

·         Now supports Print Preview capabilities. This allows the end-users to see a preview of the printout.  Footers and Headers as well as page numbers can be seen in the print preview as well.

·         Supports Microsoft RichEdit version 4 by setting the MSVersion property to a value of 4. This allows greatly improved table support including wrappable table cells.

·         New RichEditBar component: The RichEditBar component allows you to have a full word-processor attached and embedded into your own form.  While the TwwDBRichEdit can bring up a pop-up word processor, the RichEditBar differs in that the word processing capabilities are available directly from your own form.

 

New ImageCombo component:

·         The new ImageCombo component has capabilities similar to the TwwDBComboBox, but it also supports images from an ImageList within both the dropdown list and the edit portion of the control. The ImageCombo can also be embedded into the InfoPower grid or datainspector component, supporting seemless and codeless integration. The ImageCombo also supports the mapping of display values to a separate stored value.

 

Grid enhancements:

·         New method to force the current record to be the top row in the grid.

·         Regular expression support and integration.

·         ImageCombo support and integration.

 

Distribution for Delphi 2007 .WIN32

If you are using the regular expression masking support, you will need to copy the file wwregex.dll and cc3270.dll files located in your \ipstudio2007\bin\windows directory to your client’s \windows\system32 directory.

 

Installing the InfoPower help file

Help files are automatically installed into the Delphi 2007 and Delphi 2006 environments.

 

For Delphi 7, you can install InfoPower’s online help by running the OpenHelp utility. To can do this, by clicking on Help | Customize from the Delphi IDE.  From there, click on Edit | Add Files to add ip2007d7.hlp (Delphi 7)to the list of help files in the index and link tab pages. Do not add the help file to the contents tab page. After adding the InfoPower help file, click on File | Save Project to save your changes.

 

Warning – Delphi’s default help indexing is near the limits of Microsoft’s WinHelp.  As a result, you may need to remove any indexes you do not regularly use do stay within the capabilities of WinHelp.  If you experience an empty list on the Index tab page of WinHelp, this indicates you need to remove some .hlp files using OpenHelp.

Note to Delphi users who are using the InfoPower TwwDBRichEditMSWord component for InfoPower Studio/Win32:

It is important that you install the correlating InfoPower RichEditMSWord package based on the Office automation package you have selected.  For instance, if you install the office xp automation package (Dclofficexp100.bpl) you should use the IPSTUDIOWINWORDXP design-time package, instead of IPSTUDIOWINWORD2000. These packages are located in your \ipstudio2007\bin\windows directory. You will also need change your uses clause in your Pascal source so that it references wwrichedspellxp, instead of wwrichedspell2000 or wwrichedspell.

 

We also recommend omitting the InfoPower IPSTUDIOWINWORD* runtime packages from your list of runtime packages for your project, as this removes the requirement to distribute both the corresponding ip*word* packages, as well as the office automation package. Omitting the these InfoPower runtime packages from your project will add about 10K to your executable size.

 

TwwRichEditBar and SpellCheck: If you wish to use the Microsoft spell checker with the TwwRichEditBar component, you will need to add code to your OnSpellCheck event to call MSWordSpellChecker.  For instance, you can add the unit wwrichedspellxp to your form’s uses clause and then call MSWordSpellChecker from your OnSpellCheck event of the TwwRichEditBar.RichEdit component.

 

procedure TMainDemo.wwRichEditBar1RichEditSpellCheck(Sender: TObject);

begin

    MSWordSpellChecker(wwRichEditBar1.RichEdit);

end;
