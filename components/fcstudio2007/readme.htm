Compatibility issues of 1stClass Studio/Win32:

1stClass Studio for Win32 is designed to be backwards compatible with 1stClass 4000 and 1stClass Studio 2006.  The only change required is for the TfcImageForm.  If you are using this component in a dialog, you will need to call the ApplyBitmapRegion method for the imageform from your form’s OnShow event.

 

fcImageForm1.ApplyBitmapRegion;

 

Without the above code, the form will not take on the shape of the image form’s bitmap.
